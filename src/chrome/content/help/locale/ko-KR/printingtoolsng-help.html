PrintingTools NG 개요

PrintingTools NG는 이메일 인쇄에 대한 많은 사용자 정의를 허용합니다. 확장에는 헤더와 메시지 부분 모두에 대한 많은 옵션이 있습니다. 또한 일반적인 Thunderbird 인쇄 미리보기에서는 수행할 수 없는 페이지 옵션을 설정할 수 있습니다. 유연한 토큰 기반 시스템을 사용하여 자동 이름 지정으로 PDF 출력도 수행할 수 있습니다.

인쇄 액세스와 옵션을 모두 활성화하려면 도구 모음에 PrintingTools NG 버튼을 추가해야 합니다. 추가 옵션은 애드온 탭의 확장 프로그램에 대한 렌치 버튼으로 액세스할 수 있습니다.

다음은 헤더 형식의 예입니다.


맞춤 날짜 형식

사용자 정의 날짜 옵션을 사용하면 이메일 헤더의 날짜 및 수신 날짜 모두에 사용되는 날짜 형식을 완전히 제어할 수 있습니다. 사용자 지정 날짜는 거의 모든 형식을 허용하는 여러 지역화된 옵션을 제공합니다. 사용자 지정 날짜 문자열에는 구분 기호, 아래 나열된 날짜\시간 토큰과 혼합된 유니코드 문자를 포함한 리터럴이 포함될 수 있습니다.

참고: 아래 예제의 사용자 지정 날짜 결과는 실제 날짜, 사용자 로케일 및 시간대에 따라 다릅니다.


예:

날짜 문자열 결과 메모
%Y%0%m%1%d%2 %a %t %H:%M:%S [GMT %z] 2022年04月20日 水 JST 16:06:45 [GMT +0900] 리터럴이 있는 ja 로케일
%Y%0%m%1%d%2 %a %t %H:%M:%S 2022년04월20일 수 KST 16:11:06 리터럴이 있는 ko 로케일
%Y%0%m%1%d%2 %a %H:%M:%S %Z 2022年04月20日 周三 18:01:42 北京时间 zh-CN 로케일, 짧은 날
%Y%0%m%1%d%2 %A %H:%M:%S %t 2022年04月20日 星期三 18:01:42 CST zh-TW 로케일, 긴 하루
%d.%m.%Y %H:%M:%S %t 20.04.2022 18:01:42 CET 도트 구분 기호
%a, %b %d %Y %H:%M:%S Mi, Apr 20 2022 18:01:42 de locale, 짧은 일/월
%A, %B %d %Y (%H:%M:%S) %t Wednesday, April 20 2022 (18:01:42) EDT en-US 로케일, 긴 일/월

사용자 지정 날짜 형식은 기능과 같은 유닉스 strftime() 과 일치하는 % 시퀀스를 사용합니다. 날짜 및 시간 형식을 지정하는 함수인 strftime() 의 JavaScript 포트를 기반으로 합니다.
TH Doan(https://thdoan.github.io/strftime/)의 strftime() 포트


전환 순서 설명

순서 설명
%a 요일의 약식 이름입니다.
%A 요일의 전체 이름입니다.
%b 축약된 월 이름입니다.
%B 전체 월 이름입니다.
%c 현재 로케일에 대한 기본 날짜 및 시간(UTC) 표현입니다.
%C 2자리 정수로 된 세기(연도/100).
%d 10진수 형식의 날짜(범위 01~31).
%e 10진수 형식의 날짜(범위 1~31).
%F ISO 8601 날짜 형식( %Y-%m-%d 와 동일).
%G 십진수로 세기를 사용하는 ISO 8601 주 기반 연도. 4자리 연도는 ISO 주 번호에 해당합니다( %V 참조). 이는 ISO 주 번호가 이전 연도 또는 다음 연도에 속하는 경우 해당 연도가 대신 사용된다는 점을 제외하면 %Y 와 형식 및 값이 동일합니다.
%g %G 와 비슷하지만 세기가 없습니다. 즉, 2자리 연도(00-99)가 있습니다.
%H 24시간제를 사용하는 10진수로 나타낸 시간(범위 00~23). %k 도 참조하십시오.
%I 12시간제를 사용하는 10진수로 나타낸 시간(범위 01~12). %l 도 참조하십시오.
%j 10진수 형식의 날짜(범위 001~366).
%k 24시간제를 사용하는 10진수로 나타낸 시간(0~23 범위). %H 도 참조하십시오.
%l 12시간제를 사용하는 10진수 형식의 시간(범위 1~12). %I 도 참조하십시오.
%m 월을 10진수로 표시합니다(범위 01~12).
%n 월을 10진수로 표시합니다(범위 1~12).
%M 10진수 형식의 분(범위 00~59).
%p 주어진 시간 값에 따라 "AM" 또는 "PM"입니다. 정오는 "PM"으로, 자정은 "AM"으로 처리됩니다.
%P %p 와 같지만 소문자("am" 또는 "pm")입니다.
%s Epoch, 1970-01-01 00:00:00 +0000(UTC) 이후의 초 수입니다.
%S 초는 10진수로 표시됩니다(범위 00~59).
%u 요일은 십진법(범위 1~7), 월요일은 1입니다 %w 도 참조하십시오.
%V 현재 연도의 ISO 8601 주 번호를 10진수(범위 01~53)로 표시합니다. 여기서 주 1은 새해에 최소 4일이 있는 첫 번째 주(즉, 첫 번째 목요일)입니다.
%w 십진수로 표시되는 요일(범위 0~6), 일요일은 0입니다. %u 도 참조하십시오.
%x 시간을 제외한 현재 로케일에 대한 기본 날짜 표현입니다.
%X 날짜가 없는 현재 로캘의 기본 시간 표현입니다.
%y 세기가 없는 10진수 연도(범위 00~99).
%Y 세기를 포함하는 십진수로 연도.
%z +hhmm 또는 -hhmm 숫자 시간대(즉, UTC로부터의 시간 및 분 오프셋).
%Z 시간대 이름 또는 약어입니다.
%t 짧은 시간대 약어.
%0 연도 리터럴. 주로 아시아 로케일용으로 현지화되었습니다. (zh-CN: 년, 코: 년)
%1 월 리터럴. 주로 아시아 로케일용으로 현지화되었습니다. (zh-TW: 月, ko: 월)
%2 일 리터럴. 주로 아시아 로케일용으로 현지화되었습니다. (ja: 日, ko: 일)

호환성 정보:

  • %c - 형식이 지정된 문자열이 약간 다릅니다.
  • %D - 구현되지 않음( %m/%d/%y 또는 %d/%m/%y 사용)
  • %e - 공백이 추가되지 않음
  • %E - 구현되지 않음
  • %h - 구현되지 않음( %b 사용)
  • %k - 공백이 추가되지 않음
  • %n - %m 과 비슷하지만 앞에 0이 없습니다(개행에 \n 사용).
  • %O - 구현되지 않음
  • %r - 구현되지 않음( %I:%M:%S %p 사용)
  • %R - 구현되지 않음( %H:%M 사용)
  • %t - 구현되지 않음( \t 사용)
  • %T - 구현되지 않음( %H:%M:%S 사용)
  • %U - 구현되지 않음
  • %W - 구현되지 않음
  • %+ - 구현되지 않음
  • %% - 구현되지 않음( % 사용)

PDF 출력 및 자동 파일 이름 지정

PDF 출력은 이제 토큰 시스템을 사용하여 자동으로 이름을 지정하고 사전 정의된 PDF 출력 디렉토리에 저장할 수 있습니다. 이는 각 이메일의 이름과 위치를 선택할 필요가 없기 때문에 선택한 여러 이메일을 인쇄할 때 특히 유용합니다.

PDF 출력 디렉토리를 활성화하고 저장 위치에 대한 디렉토리를 선택하십시오. 출력 파일 이름은 구성된 파일 이름 형식을 따릅니다.

파일 이름 형식은 토큰과 구분 기호 또는 공백과 같은 정적 문자로 구성됩니다. 확장된 파일 이름 형식은 토큰을 사용합니다. 정확한 파일 이름은 토큰과 텍스트 또는 유효한 파일 이름 문자를 사용하여 구성할 수 있습니다. 잘못된 문자는 다음과 같습니다. | * : < > ? \ / "

기본 이름 형식은 이메일의 제목인 ${subject}입니다.

예

${subject} - 기본 형식, 이메일 제목
${prefix}_${date_custom}_${sender} - 기본값 대신 사용자 정의 날짜 형식 사용
${subject}-GmailMsg-${date}-${index} - 맞춤 텍스트 포함

아래 표는 파일 이름 형식에서 사용할 수 있는 전체 토큰 세트를 설명합니다.

토큰 설명
${subject} 메시지의 제목입니다.
${sender} 메시지 발신자의 이름입니다.
${recipient} 메시지 수신자의 이름입니다.
${sender_email} 메시지 보낸 사람의 전자 메일 주소입니다.
${recipient_email} 메시지 수신자의 이메일 주소입니다.
${smart_name} 메시지가 보낸 편지함 폴더에 있으면 받는 사람, 그렇지 않으면 보낸 사람 이름
${index} 메시지 인덱스 번호입니다.
${prefix} 접두사 텍스트 옵션 값입니다.
${suffix} 접미사 텍스트 옵션 값입니다.
${date} 표준 날짜 형식입니다. 기본 형식은 YYYYMMDDHHmm입니다.
${date_custom} 사용자 지정 날짜 형식입니다. 사용자 정의 날짜 형식 섹션을 참조하십시오.

인쇄 및 페이지 옵션

이 옵션을 사용하면 프린터 페이지별 옵션을 설정할 수 있습니다. 미리보기 대화 상자가 표시되지 않고 인쇄가 즉시 수행되는 무음 모드를 설정할 수도 있습니다.

페이지 옵션을 사용하면 미리 보기 또는 인쇄가 모두 완료될 때 항목을 설정할 수 있습니다. 미리보기 대화 상자에서 설정할 수 없는 일부 항목에는 머리글과 바닥글이 포함됩니다. 다른 항목은 미리보기에서 유지되지 않으며 일반적으로 인쇄가 완료될 때마다 설정해야 합니다.

여기에는 사본 및 페이지 범위가 포함됩니다. 일반적인 사용 사례는 이메일의 첫 페이지만 기본값으로 인쇄하려는 경우입니다. 일반적으로 매번 페이지 범위를 변경해야 합니다.

노트:

페이지 범위는 미리보기에서와 같은 방식으로 설명됩니다. 예는 1, 1-2, 3-4,5입니다. 미리 설정된 페이지 범위가 지정된 메시지에 유효하지 않은 경우 미리보기에 오류가 표시됩니다.

여백은 사용자 로캘에 적합한 단위(인치 또는 밀리미터(in 또는 mm))를 사용하여 설정할 수 있습니다.

머리글과 바닥글은 아래 나열된 다양한 메시지 변수를 포함하도록 설정할 수 있습니다. 또한 임의의 텍스트도 포함될 수 있습니다.

  • &D = 날짜(저장 시)
  • &P = 저장된 파일의 페이지 번호
  • &PT = 총 페이지 번호("3/5")
  • &T = 페이지 제목(메시지 제목)
  • &U = URL(메시지의 URL)
